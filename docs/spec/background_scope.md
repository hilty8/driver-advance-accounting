# 背景・目的・スコープ

## ドキュメント情報

- 名称: Driver Advance Accounting 仕様書
- 対象: 運営会社 O / 事業者 C / ドライバー D / 開発者
- 目的: プロジェクト全体の要件・用語・業務フロー・主要ロジックを統合し、認識差分を解消する。
- スコープ: 要件定義レベル（実装詳細は別途設計書）

## 背景

- D は支払サイトが長いことがあり、前借りニーズが強い。
- C は前借り管理を手作業で行っており、負荷やミスが発生しやすい。
- O は前借り管理をシステム化し、運用の効率化と収益化を図る。

## 目的

- D がスマホから前借り申請できるようにする。
- C が前借り残高や支払予測を一元管理できるようにする。
- O が利用状況と収益を把握・管理できるようにする。

## スコープ

### In Scope

- O/C/D の 3 者を前提とした前借り管理機能
- 前借り元本・手数料・未回収残高の管理
- 月間利用額・手数料収入・回収額・回収率の集計
- CSV インポート/エクスポート
- 支払日を起点とした日次バッチ処理

### Out of Scope

- 給与/会計システムのリアルタイム連携
- 厳密な与信・遅延/延滞のリスクモデル
- KYC/反社チェックなどの法的審査
- 銀行振込 API の実装（V1 では手動振込を許容）

## 役割と権限（概要）

- O: テナント管理、監査/ログ確認、SLA 監視、請求監視。
- C: ドライバー管理、日報取り込み、前借り承認、給与/回収調整。
- D: 前借り申請、入金予定/ステータス確認、通知確認。
